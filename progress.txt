- Stories are dependency ordered; document planned commands explicitly when their workspace/tooling lands later, and do not invent unconfirmed configuration defaults.
- Dependency installation, dependency changes, configuration changes, and database migrations require confirmation before execution.
- Local Compose data services bind to `127.0.0.1`; PostgreSQL is the only default service and future-only services must use explicit profiles.
- Project-local skills live under `.opencode/.opencode/skills`; they reference root `AGENTS.md`, use Bun scripts with Vitest, and report unavailable later-story tooling instead of creating it early.
- Workspace packages use the `@gamenight-bingo/*` namespace, extend `tsconfig.base.json`, and expose only compileable source entry points until their dependency-ordered implementation stories land; root workspaces are restricted to exactly `apps/*` and `packages/*`, and new package directories under them must be added to the validated workspace contract.
- Run `bun run test:workspace-boundaries` after workspace-manifest or domain-import changes; the TypeScript AST guard checks static, type-only, dynamic, direct CommonJS (including TypeScript-transparent and comma loader expressions), JSDoc, and triple-slash references with normalized path separators without matching ordinary comments or strings.
- Boundary parsing must use the source file's actual script kind and fail closed on syntax errors; workspace checks also require real `tsc` scripts without strictness downgrades, prohibit shared/domain module aliases, `rootDirs`, and `moduleSuffixes`, constrain relative source and root/nested package-resolution targets to `packages/domain/src`, and reject source/reference symlinks across TypeScript declaration and JavaScript-extension resolution candidates plus escaping absolute, file-URL, relative, type-reference, file-reference, and AMD dependency paths.
- Workspace typechecks must cover every discovered `src` file without inherited or implicit exclusions, verified against TypeScript's parsed effective file set, and must reject all parsed configuration diagnostics while retaining effective `noEmit` with `outDir`/`declarationDir` outside `src`; domain source also rejects JavaScript and `@ts-nocheck`. Shared/domain TypeScript configuration cannot inherit or synthesize implicit module imports, and domain bare imports plus package-valued type/AMD directives must be Node built-ins, self-references, or declared dependencies without traversal segments and validated against forbidden runtime or type-package aliases.
- Keep intentional ESLint, Prettier, TypeScript, Vitest, React Testing Library, and Playwright failures under `tests/fixtures/quality`; normal checks exclude them, while `bun run test:quality-tooling` requires each tool to fail for its expected diagnostic rather than accepting any nonzero exit.
- Vitest discovery is limited to repository `apps`, `packages`, and `tests` paths, explicitly excludes nested `node_modules` reached through Bun workspace links, runs TS tests in Node and TSX tests in jsdom, and leaves `tests/e2e` to the separate `bun run test:e2e` command.
- Runtime entry points pass `process.env` to `parseRuntimeConfig` from `packages/contracts`; downstream and domain code consume the immutable parsed type rather than reading environment state directly, and startup subprocess tests isolate environment/module state.
- After workspace dependency manifests change, run `bun install` to refresh `bun.lock`, then verify reproducibility with `bun install --frozen-lockfile`.
- Versioned Zod wire contracts live under `packages/contracts/src/v1`; every nested object is strict, top-level messages carry `schemaVersion`, identifiers and active-lobby sequences are branded, client commands contain intent rather than authoritative state, and authorized snapshots structurally omit credentials, foreign cards, future draws, prior-round history, and active event history. Lobby-wide sequences belong only on messages delivered to every authorized lobby consumer; private-only events omit them, acknowledgements declare their scope, and snapshot refinements enforce cross-object roster/state consistency.
- `docs/bingo-pattern-catalog.md` inventories source diagrams as `p1/dNN` in reading order with exact, flexible-example, or alias modes and tested 5x5 source masks; its generated core section is parity-tested from `packages/patterns/src/catalog.ts` rather than maintaining a second runtime mask source.
- Domain randomization accepts injected `CryptographicRandomBytes`; runtime adapters own the secure byte source, deterministic tests inject seeded bytes, and bounded integer selection uses rejection sampling rather than biased modulo reduction.
- Runtime pattern definitions store every accepted required-cell arrangement as a 5x5 mask: One Line has 12 line masks, Two Lines has all 66 distinct line-pair unions, exact patterns retain one untransformed source mask, and matching always satisfies index 12 as the free center.
- Exact generated Markdown parity blocks use `prettier-ignore` markers around the generated boundaries so formatting cannot silently rewrite canonical generator output.
- Source review records map every PDF diagram to its runtime ID and disposition (`exact-mask-match`, `flexible-rule-example`, or `source-alias`), record all 25 cells reviewed, and parity-test exact records against canonical runtime masks without duplicating masks in the review table.
- Pattern preview galleries derive a normalized model from `patternCatalog`; flexible-rule source examples live with the canonical pattern source, committed JSON goldens cover every runtime selection, and standalone generated HTML is parity-tested and isolated from Prettier with an exact-node ignore marker.
- Category preview pages reuse the source-filtered canonical preview model and shared renderer while exposing category-specific creation/generation functions, committed goldens, and standalone HTML paths.
- Complete pattern audits join all PDF inventory/review rows to runtime IDs, aliases, exact or flexible mask parity, approved identity/mask constraints, and the combined category preview goldens; mutation tests must prove each audit failure category remains fail-closed.
- Domain transitions return discriminated stable errors for expected state rejection and throw `RangeError` only for malformed internal values such as invalid counts, timestamps, or durations.
